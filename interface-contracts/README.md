## 三、Schema 版本兼容性说明

### v1.0（初始版本）

- 根字段：`handoff_package:`
- 结构：`deliverables / metadata / next_steps`
- 状态：**已弃用**（仅 council-to-execution.yaml 使用，待升级）

### v1.1（当前版本 · v1.5.0 引入）

- 根字段：`handoff:`
- 结构：`schema_version / from_skill / to_skill / payload / user_action / created_at`
- 新增可选字段：`self_review`（执行前假设 + 潜在失败点 + S05 预扣分预测）
- 新增可选字段：`downstream_notes`（对下游 Skill 的注意事项 + 必须验证项）
- 状态：**当前标准**

### 交接包命名体系（HP-D/A/B/C/E/F）

| 交接包 | 对应文件 | 传递方向 | 核心内容 |
|---|---|---|---|
| HP-D | s00-to-any.yaml | S00 → 任意 | 意图类型、置信度、路由推荐、用户原文 |
| HP-A | s01-to-s02.yaml | S01 → S02 | 三档提示词、质量评分、平台约束、温度建议 |
| HP-B | s03-to-s02.yaml | S03 → S02 | 评估矩阵、选定方案、缺口列表、配置建议 |
| HP-C | s02-to-s05.yaml | S02 → S05 | 工程包元数据、测试目标、已知限制、设计决策 |
| HP-E | s04-to-user.yaml | S04 → 用户 | 环境信息、前提条件、分步操作、断点恢复 |
| HP-F | s05-to-release.yaml | S05 → 发布 | 验收结论、测试结果、风险评估、上游反馈包 |

## 四、使用注意事项

**读取顺序**：使用交接包时，请按以下顺序读取 YAML 文件字段：
1. 确认 `schema_version`（确保与当前体系版本兼容）
2. 确认 `from_skill` 和 `to_skill`（确保数据流向正确）
3. 读取 `payload`（核心传递数据）
4. 检查 `self_review`（可选，了解上游 Skill 的执行预警）
5. 检查 `downstream_notes`（可选，了解需要特别验证的内容）

**格式约定**：
- 所有 YAML 文件使用 2 空格缩进
- 注释行（`#`）说明字段用途
- `[必须用户核实]` 标注的字段不可由 AI 填写确定性数据
- 空字段保留占位符而非删除（便于复用模板）
