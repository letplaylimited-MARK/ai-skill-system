SKILL 00
Navigator · 智能体导航官

›

核心价值

不知从哪开始时的第一个入口，消除选择困难，智能路由至最合适的 Skill 组合

使用时机

  • 有模糊想法，不确定用哪个 Skill

  • 输出交接包A + pipeline 全链路规划

SKILL 03
Scout · 开源侦察官
L1 理解加工层
›

核心价值

用七维度科学评估代替「凭感觉选库」，提供有数据支撑的技术选型方案，避免选型失误

使用时机

  • 需要找开源库 / 框架 / 工具
  • 不知道用什么技术栈
  • 需要系统性技术评估报告

核心能力

  • REQ-XXX 需求拆解标准化
  • 七维度评估矩阵：功能30% / 易用20% / 性能15% / 可维护10% / 社区10% / 兼容10% / 文档5%
  • 幻觉防护（Star数/版本号标注，需用户核实）
  • 降级路径：找不到时给出 3 条备选
  • 4阶段流程：需求理解→开源搜索→七维评估→交接包输出

激活词

找开源库 技术选型 用什么框架 推荐工具

输出物

选型报告 + 交接包B（含 selected_skills / sop_hint 模块建议）

SKILL 04
Planner · 执行规划官
L2 构建层
›

核心价值

将抽象的 SOP 转化为任何人都能按步骤执行的操作手册，消除「知道做什么但不知道怎么做」的困境

使用时机

  • 有 SOP 或需求但不知如何落地
  • 需要生成详细部署步骤
  • 需要项目执行计划与时间估算

核心能力

  • P-T-S 三层结构（Phase / Task / Step，编号 P1-T2-S3）
  • 每个 Step 含：工具 / 操作 / 预期结果 / 验证 / 估时 / 依赖
  • PERT 三点估时（乐观 / 正常 / 悲观）
  • 断点恢复机制（.project_state 状态文件）
  • 三不原则：不过度设计 / 不静默调整SOP决策 / 不跳过验证

激活词

执行计划 操作手册 怎么落地 生成步骤

输出物

完整 Markdown 操作手册 + 交接包C（含 test_targets / total_phases）

SKILL 05
Validator · 测试验收工程师
L3 验收层
›

核心价值

发布前的最后质量门禁，用量化评分取代主观判断，缺陷模式归档驱动体系持续改进

使用时机

  • 项目 / Skill 完成后上线前
  • 需要量化质量评估报告
  • 发布决策需要数据支撑

核心能力

  • 五维度量化评分：功能40% / 文档25% / 可执行性20% / 接口10% / 安全5%
  • P0-P3 缺陷分级（P0 阻塞 → FAIL）
  • 三档发布决策：PASS(≥80%,P0=P1=0) / CONDITIONAL_PASS(≥70%,P1≤3) / FAIL
  • 飞轮机制：缺陷归档 → 上游 Skill 改进建议

激活词

验收 质量检查 发布前检查 可以发布了吗

输出物

验收报告 + 交接包E（含 decision / composite_score / flywheel 改进建议）
