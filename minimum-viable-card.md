# 体系精髓卡 · Minimum Viable Card
> 如果你只能记住一件事：**AI 不是一个全能工具，而是六个专职角色——每次只激活一个，给它清晰的输入，拿到标准的输出，然后搬运到下一个。**

---

## 三条核心规则

**规则 1 · 每次只激活一个角色**

不要在同一个对话里让 AI 既选型又规划又验收。一个角色做好一件事，然后用「交接包」把结果传给下一个角色。

**规则 2 · 交接包是连接器，不是可选项**

每个 Skill 的输出都应该生成一个 YAML 格式的「交接包」。复制它，粘贴到下一个 Skill 的对话开头——这是信息不丢失的唯一保证。

**规则 3 · 不确定先走 Skill 00**

不知道该用哪个 Skill？不知道从哪里开始？直接激活 Skill 00（Navigator），它会帮你分析意图并规划路径。

## 六角色速查

```
Skill 00 · Navigator   → 「我不知道从哪里开始」
Skill 01 · 提示词工程师 → 「我要设计/优化一个系统提示词」
Skill 02 · SOP 工程师  → 「我要把想法打包成标准 Skill 工程包」
Skill 03 · Scout       → 「我要选一个开源工具/框架」
Skill 04 · Planner     → 「我要把项目拆解成可执行的步骤」
Skill 05 · Validator   → 「我要验收一个 Skill 或方案是否达标」
```

## 完整流程（60秒读懂）

```
你的需求
   │
   ▼

   │  ⤷ 记忆锚贯穿整个执行链路（每个 Skill 前自动回顾约定与风险）
   ▼
Skill 00 · Navigator

   │
   ├─── 需要选工具？────→ Skill 03 · Scout

   │                         │         │
   └─── 需要执行规划？────────┤         │
                              ▼         ▼
                         Skill 02 · SOP 工程师
                         「10阶段打包 → 交接包 D」
                              │
                              ▼
```

## 五种使用模式速查

| 模式 | 适用场景 | 操作步数 | 时间成本 |
|---|---|---|---|
| **A · 单 Skill** | 已知需要哪个能力，任务边界清晰 | ~5 步 | 最低 |
| **B · 协调官入口** | 不确定从哪开始，需要动态切换角色 | ~8~12 步 | 中 |
| **C · Pipeline 流水线** | 完整项目开发流程（从想法到上线） | ~20~30 步 | 较高 |
| **D · 圆桌 + 单 Skill**（v1.5.0） | 明确任务但想提前识别风险 | ~8~10 步 | 中低 |
| **E · 深度精炼**（v1.5.0） | 重要项目、需要留下完整记录 | ~40~60 步 | 最高 |

**模式选择决策树**：
```
任务是否明确单一 Skill？
  是 → 模式 A
  否 → 是否是完整项目开发？
    是 → 是否需要高质量 + 完整记录？
      是 → 模式 E（深度精炼）
      否 → 模式 C（Pipeline）
    否 → 是否想预检风险？
      是 → 模式 D（圆桌 + 单 Skill）
      否 → 模式 B（协调官入口）
```
